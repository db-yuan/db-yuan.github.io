Notes: An Introduction to Hypothesis Testing

The basic idea

Hypothesis testing is like a “probability” version of proof by contradiction. It’s based on the principle of small probability (we’re assuming that a small probability event will not occur in an experiment). When we do hypothesis testing, we usually first state a hypothesis about the population that contradicts our expected result, then use sample data to construct a test statistic, check if the small probability event occurs, and test to what extent we can say that the hypothesis holds.

Some concepts

Two types of hypotheses

  1. Null hypothesis (H0H_0H0​): a claim about the population that contradicts our desired result. We assume that H0H_0H0​ is true until we reject it. It usually uses symbols ===,≤\le≤, or ≥\ge≥.
  2. Alternative hypothesis (H1H_1H1​): the complimentary of H0H_0H0​. It usually uses symbols ≠\ne​=,>>>, or <<<.

Two types of errors

  1. Type Ⅰ error (α\alphaα error): the error that we reject H0H_0H0​, but H0H_0H0​ is true.
  2. Type Ⅱ error (β\betaβ error): the error that we fail to reject H0H_0H0​, but H0H_0H0​ is false.

The probability of different cases:

H0H_0H0​ is true H0H_0H0​ is false
reject H0H_0H0​ α\alphaα (type Ⅰ error) 1−α1-\alpha1−α (correct decision)
fail to reject H0H_0H0​ 1−β1-\beta1−β (correct decision) β\betaβ (type Ⅱ error)

We can’t reduce both α\alphaα and β\betaβ (two probabilities) at the same time without increasing the sample size, so we should control the one that will cause a more serious consequence. In practice, people usually set a significant level α\alphaα for the type Ⅰ error, and by doing this it means that “first I have to control the chance of making a type Ⅰ error under some level (say 5%5\%5%), and then try to reduce the chance of making a type Ⅱ error, but I can’t quantify anything for the latter part.” Accordingly, we’ll consider the result that we hate, or that will cause more damage, as our null hypothesis, so at least we can use numbers to describe the chance of a bad consequence, and this is also why hypothesis testing is like proof by contradiction. Here are two unstrict examples of H0H_0H0​ (take α=5%\alpha=5\%α=5%):

  1. H0H_0H0​: the medicine is poisonous. Hence when rejecting H0H_0H0​, we can claim that the probability that “it’s poisonous but we think it’s not and take the medicine” is less than 5%5\%5%.
  2. H0H_0H0​: the new design of the app can’t bring more customers. Hence when rejecting H0H_0H0​, we can claim that there is only less than 5%5\%5% chance that the new design is useless.

Two types of tests

  1. One-tailed test: If H0H_0H0​ has the form μ=μ0\mu=\mu_0μ=μ0​.
  2. Two-tailed test: If H0H_0H0​ has the form μ≥μ0\mu\ge\mu_0μ≥μ0​ or μ≤μ0\mu\le\mu_0μ≤μ0​.

Test statistic

A test statistic is a random variable calculated from sample data. In hypothesis testing, we assume that this random variable follows some appropriate distribution. By considering its probability density function and the observed value we get, we can tell if a small probability event occurs and then decide whether we should reject H0H_0H0​ or not.

Significant level and critical value

The significant level is the value we set for α\alphaα, or equivalently it’s the bound we set for the chance of making a type Ⅰ error. The critical value is the value of the test statistic which corresponds to a given significant level.

PPP-value

We can consider the ppp-value as the probability of getting the observed value of the test statistic that we choose.

Rejection region

The rejection region is an area of the graph of the probability density function where we should reject H0H_0H0​. It’s determined by the test statistic and the significant level.

The logic of hypothesis testing

  1. State the null hypothesis and the alternative hypothesis.
  2. Select and calculate the test statistic according to the problem.
  3. Set a significant level α\alphaα and calculate the ppp-value.
  4. By comparing two values in step 3 to decide whether we reject (if p<αp<\alphap<α) or fail to reject (if p>αp>\alphap>α) the null hypothesis.

When p>αp>\alphap>α we can’t say that we “accept” H0H_0H0​ since we have assumed that H0H_0H0​ is true and are trying to see if we can reject it.
